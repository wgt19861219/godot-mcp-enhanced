# Godot MCP Enhanced 审查与改进建议（2026-05-02）

本文基于源码静态审查与本地构建结果，总结当前插件的主要问题与可改进方向。

## 总体结论

- 这个 MCP 插件功能面覆盖很广（执行、解析、验证、资源管理、文档检索），对 AI 闭环开发很有价值。
- 当前最明显的问题不在“能不能用”，而在：**路径安全/路径语义一致性**、**单文件复杂度过高**、**跨平台健壮性与可观测性**。
- 建议优先处理路径相关问题（P0），随后做架构拆分（P1）。

---

## 发现的问题

### P0：`validatePath()` 与 `join()` 混用存在路径语义风险

`validatePath()` 会把相对路径 `resolve()` 成绝对路径；但一些调用点又把它作为“子路径”拼接：

```ts
const target = subdir ? join(p, validatePath(subdir)) : p;
```

当 `validatePath(subdir)` 已是绝对路径时，`join(p, abs)` 的前缀 `p` 会被丢弃，导致结果不再位于 project 根目录下。

此外，`validatePath()` 当前只用 `p.includes('..')` 判断 traversal：

- 误报：合法文件名包含 `..` 也会失败。
- 漏报：符号链接、规范化前后路径等情况未覆盖。

**建议：**

1. 将“项目根路径校验”和“用户子路径校验”分成两个函数：
   - `resolveProjectRoot(path)`：仅解析项目根。
   - `resolveWithinRoot(root, userPath)`：`resolved = resolve(root, userPath)`，再校验 `resolved.startsWith(root + sep)`（或用 `path.relative` 判定）。
2. 对所有文件读写入口统一走 `resolveWithinRoot`，避免工具间校验策略不一致。

---

### P1：`GodotServer.ts` 过大，职责过于集中

`GodotServer.ts` 承担了：工具注册、进程管理、路径校验、场景/脚本读写、验证、截图、资源读写、文档查询等多职责，后续维护成本高，回归风险大。

**建议：**

- 按领域拆分 handler：
  - `tools/runtime-tools.ts`（run/stop/output/screenshot）
  - `tools/scene-tools.ts`
  - `tools/script-tools.ts`
  - `tools/validation-tools.ts`
  - `tools/docs-tools.ts`
- 保留 `GodotServer.ts` 只做注册和依赖组装。
- 每个 handler 内部引入轻量参数 schema（zod/自定义）做输入校验。

---

### P1：Godot 可执行文件自动发现可进一步增强

`COMMON_PATHS` 包含 Windows 通配符字符串（如 `Godot_v4*.exe`），但 `existsSync` 不会展开 glob，实际不会命中。

**建议：**

- Windows：扫描常见目录并匹配文件名正则（`/^Godot_v4.*\.exe$/`）。
- Linux/macOS：优先 `which godot4/godot`，再回退固定路径。
- 找不到时返回“已尝试路径列表”，提升可诊断性。

---

### P2：临时文件与并发清理策略可更稳健

`gdscript-executor.ts` 会在公共 temp 目录下清理旧文件。当前逻辑在并发多请求下通常可工作，但缺少“按会话隔离”的目录策略。

**建议：**

- 使用 `mkdtemp` 为一次执行创建会话目录。
- 只清理本会话目录；后台定时任务再清理全局过期目录。

---

### P2：可测试性有改进空间

目前构建通过，但仓库缺少自动化测试脚本（如 `npm test`）。

**建议：**

- 最小单测集：
  - 路径解析与越界防护。
  - GDScript marker 解析。
  - `.tscn` 解析与编辑边界条件。
- 集成测试（可选）：用最小 Godot 样例工程执行一次 `run_and_verify` 冒烟。

---

## 优先级改进路线（建议）

1. **第 1 周（P0）**：重构路径解析与越界校验，统一所有读写入口。
2. **第 2 周（P1）**：拆分 `GodotServer.ts`，引入模块化 handler。
3. **第 3 周（P1/P2）**：增强 Godot 自动发现、改进错误消息。
4. **第 4 周（P2）**：补齐核心单测与 1 条集成冒烟测试。
